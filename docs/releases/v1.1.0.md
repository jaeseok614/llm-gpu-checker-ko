## AI Hardware Fit v1.1.0 — Multi-workload AI Hardware Planner

내 GPU에서 생성형 LLM뿐 아니라 임베딩·리랭커·OCR/VLM까지 무엇을 얼마나 돌릴 수 있는지 확인하는 계산기입니다.

**[웹에서 바로 사용하기](https://jaeseok614.github.io/llm-gpu-checker-ko/)**

### 이번 릴리스 핵심

- **로컬 실측 벤치마크 CLI** (`scripts/benchmark-cli.mjs`) — Ollama/llama.cpp에 실제 프롬프트를 보내 prefill/decode 속도, TTFT, peak VRAM을 측정하고 제보용 JSON을 바로 만들어 줍니다.
- **예상 vs 실측 오차 표시** — 제보된 실측값이 현재 화면과 정확히 일치하면 상세 패널에 "예상 X vs 실측 Y · 오차 ±Z%"를 보여줍니다.
- **여러 GPU에 여러 모델 동시 배치 (베타)** — 검색 가능한 체크리스트로 생성형 LLM·임베딩·리랭커·OCR/VLM을 섞어서 선택하면 GPU별 배치와 병목 없는 동시 인원/처리량을 계산합니다.
- **동시 처리 용량 (베타)** — GPU·모델·양자화·컨텍스트 조합의 권장/최대 동시 인원과 처리량을 KV cache 기준으로 역산합니다.
- **모델 라이선스 안내** — 상업 이용 가능/조건부/비상업·연구용/원문 확인 필요를 모델별 원문 링크와 함께 표시합니다.
- **이기종 GPU 병렬** — 서로 다른 GPU 두 대의 VRAM·대역폭·연산량을 합산하고 통신 손실을 보수적으로 반영합니다.
- 리랭커·LLM·임베딩·OCR/VLM 모델 **164종에 출처 연결 공개 품질 벤치마크** 수치 추가
- 임베딩 60종, 리랭커 34종, OCR/VLM 45종 신규 추가

### 현재 지원 규모

| 항목 | 수량 |
| --- | ---: |
| GPU 프리셋 | 90 |
| 전체 AI 모델 | 286 |
| LLM / 임베딩 / 리랭커 | 147 / 60 / 34 |
| OCR / 문서 VLM / 범용 VLM | 4 / 9 / 32 |

### 실측 벤치마크 참여

속도 추정값의 신뢰도를 높이려면 실제 실행 데이터가 필요합니다.

```bash
node scripts/benchmark-cli.mjs --runtime ollama --model qwen3:8b --context 8192
```

측정 결과를 [Benchmark report 이슈](https://github.com/jaeseok614/llm-gpu-checker-ko/issues/new?template=benchmark-report.yml)로 제보해 주시면 반영하겠습니다.
